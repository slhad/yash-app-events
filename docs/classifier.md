# Generic ONNX classifier

The classifier backend loads a portable profile-relative ONNX model on CPU through
`ort`/ONNX Runtime. Profiles declare the model SHA-256, ordered output labels, input
dimensions, deterministic preprocessing, change trigger, and forced refresh interval.

Before session creation, validation rejects absolute/traversing model paths, malformed
hashes, duplicate or excessive labels, excessive input dimensions, invalid scheduling,
non-files, files over 64 MiB, and digest mismatches. After inference, output cardinality
must exactly match the declared labels. Failures produce an error observation rather
than a negative classification.

## Dataset and model

`crates/vision/tests/fixtures/classifier` is generated by
`scripts/generate-classifier-fixtures.py`. It contains eight noisy, shifted 8×8 HUD
icon images split between `orb` and `cross`, a generated 765-byte ONNX model,
and a manifest containing labels, expected cases, dimensions, license, and model hash.
No downloaded weights or copyrighted game images are used.

Regenerate with an isolated pinned environment:

```bash
python -m venv /tmp/yash-fixtures
/tmp/yash-fixtures/bin/pip install -r scripts/requirements-fixtures.txt
/tmp/yash-fixtures/bin/python scripts/generate-classifier-fixtures.py
```

The tiny model intentionally proves deployment contracts rather than claiming broad
visual intelligence. It flattens normalized pixels and applies a two-output linear
nearest-prototype classifier for two semantic synthetic item icons. Real character,
pet, or game-mode models require their own representative datasets, hashes, benchmarks,
and profile assets.

## Reference benchmark

Run:

```bash
cargo run --release -p yash-app-events-vision --example classifier_benchmark
```

The 2026-07-11 Ryzen 7 5800X3D run evaluated 80,000 cases with 100% exact accuracy,
0.534 mean confidence, 0.00333 ms mean inference latency, 26 process CPU ticks
(`CLK_TCK=100`), and 26,792 KiB process high-water RSS. Raw output is recorded in
`docs/classifier-benchmark.json`.

Live and image-replay paths share the typed detector and rule coordinator. Unchanged
processed crops reuse the prior classification until the configured change threshold
or forced refresh interval is reached. The GUI exposes model path/hash, labels, input
shape, preprocessing, scheduling, frozen diagnostics, and text-rule authoring.

A clean-prefix `scripts/install-user.sh` acceptance used the installed daemon and
protocol-v1 client under isolated XDG directories, created the hashed classifier
profile, and evaluated `orb_0.png` followed by `cross_0.png`. The installed binary
returned typed `orb`/`cross` observations and a matching `cross_icon entered` event
with precision and recall 1.0, proving the packaged ONNX Runtime path does not depend
on Cargo's test environment.
